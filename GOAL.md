# Project Goal

Stop building the todo app — it's a solved problem and not the point of this experiment.

Build something genuinely novel or interesting instead — not a clone of an existing 
common tool (todo apps, note apps, calculators, weather apps, etc). Think of something 
a bit unusual, creative, or experimental. It's fine if it's playful, weird, or niche — 
originality matters more than "usefulness" here.

Before building anything, write your idea and reasoning in DECISIONS.md: what it is, 
why it's interesting, and how you'll verify it works incrementally.

Some directions to consider (pick one, or come up with something better):
- A generative system (text, art, music, procedural content) with unexpected rules
- A simulation of something (an ecosystem, an economy, a made-up physical system)
- A tool that transforms/analyzes data in a surprising way
- A small game with an unusual mechanic
- Something that combines two unrelated ideas in a novel way

## Rules
- Work in small increments, one focused change per run
- Always run tests before committing
- Keep STATE.md updated with what you did and why
- Keep DECISIONS.md updated with reasoning for major choices
- If something is broken, fix it before adding new features
- Do not revert to a todo/task manager or similarly common CRUD app
