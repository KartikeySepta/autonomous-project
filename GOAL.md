# Project Goal

Build something small but genuinely useful. You choose what — pick 
something that can be built incrementally, has a clear way to verify 
it works (tests), and could realistically become a usable tool within 
a few weeks of hourly iterations. Explain your choice and reasoning 
in DECISIONS.md before you start building.

## Rules
- Work in small increments, one focused change per run
- Always run tests before committing
- Keep a STATE.md file updated with what you did and why
- Keep a DECISIONS.md file explaining your reasoning for major choices
- If something is broken, fix it before adding new features
